# Syamil Asyraf Portfolio

A React + TypeScript + Vite portfolio for Syamil Asyraf Bin Mohd Ali Jaafar.

The design direction is professional, futuristic, and energetic: a deep maroon base with red, cyan, and orange accents. Circles, hexagons, and angled edges are used as UI language for badges, buttons, category markers, and cards rather than as a standalone illustration.

## Local Development

```bash
npm install
npm run dev
```

For Windows PowerShell, use `npm.cmd` if script execution policy blocks `npm`:

```bash
npm.cmd run dev
```

## Build Checks

```bash
npm run lint
npm run build
```

## Project Structure

```text
src/
  components/       Reusable page sections and section helpers
  data/             Portfolio content, projects, skills, timeline data
  styles/           Fonts, design tokens, global CSS, page layout CSS
  assets/           App image assets used by React imports
Assets/             Custom font files supplied for the visual identity
public/             Static favicon and public SVG assets
```

## Editing Content

Most portfolio text lives in `src/data/portfolio.ts`.

Update this file when you want to add:

- New projects
- Better project descriptions
- Individual showcase page details
- Skills and tools
- Education or event timeline items
- Contact/profile details

The components read from that data file, so future additions usually do not require changing layout code.

## Project Showcase Pages

Project cards link to hash routes such as:

```text
#/projects/taskomon
#/projects/red-flagger
#/projects/while-play-tower-defense
#/projects/kingdom-of-gluttony-alpha
```

These routes are handled inside `src/App.tsx`, which keeps the site compatible with GitHub Pages without adding a router dependency.

Each project page is generated by `src/components/ProjectShowcasePage.tsx` from the matching entry in `src/data/portfolio.ts`.

For each project, add or improve:

- Screenshots or demo recordings in the visual proof area
- Clear feature evidence
- Build process notes
- Tools and stack
- Next improvements
- Repository, demo, or documentation links

## Project Images

Project screenshots are loaded in `src/data/assetManifest.ts` from these folders:

```text
src/assets/Taskomon
src/assets/While Play TD
src/assets/Kingdom of Gluttony
src/assets/Jumpy Beaw
```

Projects without image folders still appear, but their showcase pages omit the screenshot gallery.

## Graphic Design Gallery

The graphic design section is generated from:

```text
src/assets/GD
```

Add `.png`, `.jpg`, `.jpeg`, or `.webp` files there and they will appear in the gallery automatically.

## Fonts

The custom fonts are loaded in `src/styles/fonts.css` from the top-level `Assets` folder:

- `RocketRinder-yV5d.ttf`
- `Revamped-X3q1a.ttf`
- `NeonFuture20Demo-L3jp3.otf`

If you rename or move the font files, update the paths in `src/styles/fonts.css`.
